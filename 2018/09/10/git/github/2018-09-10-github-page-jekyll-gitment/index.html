部署github page：jekyll评论整合gitment

发表于 2018-09-10 | 分类于 github-page | | 阅读次数:

本文介绍了整合gitment作为评论系统。

1. 申请一个Github OAuth Application

Github头像下拉菜单 > Settings > 左边Developer settings下的OAuth Application > Register a new application，填写相关信息。

  • Application name, Homepage URL, Application description, 根据实际情况填写
  • Authorization callback URL 一定要写自己Github Pages的URL
    • 要么是xxx.github.com或者xxx.github.io，要么是自定义域名
    • 这里我的是自定义域名：github.wangkaimin.com
  • 填写完上述信息后按Register application按钮，得到Client ID和Client Secret

2. 在jekyll博客调用gitment

1、为了统一目录结构，在_includes目录下创建gitment.html

2、注意我这里使用了jekyll提供的全局变量的使用方法，_data目录下创建github.yml

3、其实这些变量直接放在gitment.html也可以的

4、在_layouts中的page.html和post.html末尾增加

5、其实也可以通过变量控制的，这里我整合了disqus，changyan，gitment，在文章中通过变量控制

3. 为每篇博文初始化评论系统

点击Initialize Comments按钮后，就可以开始对该篇博文开始评论了， 同时也可以在对应的github仓库看到相应的issue。

注意

  • 一个一个手动初始化确实很麻烦，如果能自动初始化就完美了
  • 注意gitment的id的长度限制，会报”Error：validation failed”错误，所以这里没有使用默认的，而是使用的了page.url
  • 引用的js和css并没有压缩，为了提高速度可以使用在线工具压缩，放在自己网站的静态资源里
